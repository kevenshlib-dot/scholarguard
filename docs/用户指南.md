## 1. 欢迎使用 ScholarGuard

### 1.1 ScholarGuard 是什么？

ScholarGuard 是一款**学术 AI 内容检测平台**，帮助您识别文本中可能由 AI（如 ChatGPT、Claude 等）生成的内容。与传统检测工具不同，ScholarGuard 采用先进的大语言模型（LLM）作为核心分析引擎，能够更精准、更全面地评估文本的原创性。

### 1.2 能帮您做什么？

| 使用场景 | 说明 |
|---------|------|
| 🔍 **论文自查** | 在提交论文前，检查您的文章是否存在被误判为 AI 生成的风险 |
| ✍️ **写作改进** | 获取智能建议，让您的学术写作更自然、更具个人风格 |
| 📋 **教学审核** | 教师可以检测学生作业中的 AI 辅助写作痕迹 |
| 📊 **期刊审稿** | 编辑可以辅助评估来稿的原创性 |

### 1.3 检测原理简介

ScholarGuard 的检测并非简单地"匹配数据库"，而是让一个大语言模型从**多个维度**分析您的文本：

- **语言模式分析**：检查句式、用词是否呈现 AI 生成的典型特征
- **结构特征评估**：分析段落组织、论证逻辑是否过于"模板化"
- **统计特征计算**：测量词汇丰富度、句长变化等量化指标
- **证据融合判定**：综合以上多项指标，给出最终的风险评估

> 💡 **重要提示**：检测结果是概率性的参考意见，不应作为唯一判定依据。ScholarGuard 提供的是"线索与证据"，最终判断权在于使用者。

## 2. 快速入门

### 三步完成第一次检测

```
第 1 步：注册账号         →  访问平台，填写基本信息即可注册
```

## 3. 账号注册与登录

### 3.1 注册新账号

1. 打开浏览器，访问 ScholarGuard 平台地址（例如 `http://localhost:3000`）
2. 在登录页面，点击右下方 **"没有账号？立即注册"** 链接
3. 填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 用户名 | ✅ | 您的显示名称，不可与其他用户重复 |
| 邮箱 | ✅ | 用于登录的邮箱地址，需唯一 |
| 密码 | ✅ | 至少 6 个字符 |
| 确认密码 | ✅ | 再次输入密码以确认 |
| 所属机构 | ❌ | 可选填您的学校或单位名称 |

4. 注意右侧的**密码强度提示**：
   - 🔴 太短 — 密码少于 6 个字符
   - 🟠 较弱 — 建议增加数字或特殊字符
   - 🟡 中等 — 可以使用但建议增强
   - 🟢 强 — 安全性良好

5. 点击 **"注册"** 按钮，注册成功后系统将自动登录并跳转到检测页面

### 3.2 登录

1. 在登录页面输入注册时使用的**邮箱**和**密码**
2. 点击 **"登录"** 按钮
3. 如果看到红色错误提示 "Invalid email or password"，请检查邮箱和密码是否正确

### 3.3 退出登录

点击左下角的**用户头像**区域，在弹出的菜单中选择 **"退出登录"**。

> 🔒 **安全提示**：在公用电脑上使用后，请务必退出登录。

## 5. AI 文本检测

这是 ScholarGuard 的**核心功能**，也是您最常使用的页面。

### 5.1 输入文本

有两种方式提交待检测的文本：

#### 方式一：直接粘贴

1. 点击左侧 **"🔍 AI检测"** 菜单
2. 在页面中央的大文本框中**粘贴**或**输入**您的文章
3. 文本框下方会实时显示**字符数**
4. 最少需要 **200 个字符**才能进行检测

#### 方式二：上传文件

1. 点击文本框上方的 **"上传文件"** 按钮
2. 支持以下文件格式：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| PDF | `.pdf` | 自动提取文本内容；扫描版 PDF 可能无法提取 |
| Word | `.docx` | 自动读取文档文本 |
| 纯文本 | `.txt` | 直接读取全部内容 |
| Markdown | `.md` | 读取原始文本 |

3. 文件上传后，提取的文本将自动填充到文本框中

> ⚠️ **注意**：如果上传的是**扫描版 PDF**（图片型），系统会弹出警告提示，此时无法提取文字内容。请改用文字版 PDF 或将内容手动粘贴。

### 5.2 选择检测参数

在文本框右侧或下方，您可以调整以下检测选项：

#### 检测粒度

| 选项 | 含义 | 适用场景 |
|------|------|---------|
| **文档级** | 对整篇文章给出一个总体评分 | 快速了解整体风险 |
| **段落级** | 逐段分析，标记每段的风险 | 定位具体有问题的段落 |
| **句子级** | 逐句分析，最精细的检测 | 需要精确定位时使用 |

#### 语言设置

| 选项 | 说明 |
|------|------|
| **自动检测** | 系统自动判断文本语言（推荐） |
| **中文** | 指定为中文文本 |
| **英文** | 指定为英文文本 |

#### 学科领域

| 选项 | 说明 |
|------|------|
| **通用** | 不限定学科，使用通用模型分析 |
| **政治学** | 针对政治学论文特征优化 |
| **经济学** | 针对经济学论文特征优化 |
| **社会学** | 针对社会学论文特征优化 |
| **法学** | 针对法学论文特征优化 |

> 💡 **建议**：大多数情况下使用默认设置即可。如果您是特定学科的论文，选择对应学科可以提高检测精度。

### 5.3 开始检测

1. 确认文本已输入且超过 200 字符
2. 点击 **"开始检测"** 按钮
3. 系统将显示检测进度，请耐心等待

检测所需时间取决于文本长度：
- 短文（< 1000 字）：通常 5-15 秒
- 中等文章（1000-5000 字）：通常 15-45 秒
- 长文（> 5000 字）：可能需要 1-2 分钟

### 5.4 阅读检测报告

检测完成后，系统将展示完整的分析报告。以下是报告各部分的详细说明：

#### 5.4.1 风险等级与总体评分

页面顶部将显示一个醒目的**风险等级徽章**和**风险评分**：

| 风险等级 | 颜色 | 评分范围 | 含义 |
|---------|------|---------|------|
| **低风险** | 🟢 绿色 | 0 - 30% | 文本很可能是人类原创 |
| **中等风险** | 🟡 黄色 | 30 - 60% | 文本存在部分 AI 特征，需关注 |
| **高风险** | 🟠 橙色 | 60 - 80% | 文本可能包含大量 AI 生成内容 |
| **极高风险** | 🔴 红色 | 80 - 100% | 文本高度疑似 AI 生成 |

> ⚠️ **重要**：风险评分并非"AI 占比"。评分 70% 不代表"70% 的内容是 AI 写的"，而是表示整体文本呈现 AI 特征的程度。

#### 5.4.2 多维度评分细节

报告中还会展示以下细分指标：

- **LLM 置信度**：大语言模型对文本是否为 AI 生成的判断信心
- **统计特征分数**：基于词汇多样性、句长变化等量化指标的得分
- **证据完整度**：检测过程中收集到的证据充分程度（进度条显示）

#### 5.4.3 段落级热力图

如果选择了"段落级"或"句子级"检测粒度，报告将包含一个**热力图**：

```
段落 1  ████████████░░░░░░░░  62%  → 中等风险
段落 2  ██░░░░░░░░░░░░░░░░░░  12%  → 低风险
段落 3  ████████████████████  95%  → 极高风险
段落 4  ████████░░░░░░░░░░░░  41%  → 中等风险
```

颜色越深表示风险越高，帮助您快速定位问题段落。

#### 5.4.4 标注文本

检测到的可疑片段会以**高亮标注**的方式显示在原文上：

- 🟡 **黄色高亮**：中等风险片段
- 🟠 **橙色高亮**：高风险片段
- 🔴 **红色高亮**：极高风险片段

#### 5.4.5 证据摘要

报告底部提供**文字说明**，解释为什么系统给出这样的判断，例如：

> "该段落使用了大量并列结构和过渡词（'首先…其次…此外…最后'），这是 AI 生成文本的常见模式。同时，词汇选择表现出高度一致的正式度，缺乏人类写作中常见的风格波动。"

### 5.5 提交反馈

如果您认为检测结果**不准确**，可以提交反馈帮助改进系统：

1. 在检测报告底部找到 **反馈区域**
2. 选择您的意见：**同意** 或 **不同意**
3. 可选：填写具体反馈说明
4. 点击提交

您的反馈将被记录，帮助系统持续优化。

### 5.6 导出到写作建议

如果检测发现了较高风险的内容，您可以一键将文本和检测结果转入**写作建议**模块：

1. 检测完成后，文本和结果会自动保存到浏览器会话中
2. 点击侧边栏 **"✍️ 写作建议"**
3. 在写作建议页面点击 **"从检测导入"** 按钮
4. 文本和检测上下文将自动加载，系统会根据检测到的问题**自动推荐优化策略**

## 6. 写作建议与优化

写作建议模块帮助您**改进被标记为有 AI 特征的文本**，使其更具个人风格和学术深度。

### 6.1 进入写作建议

有两种方式进入：

1. **从检测页面导入**（推荐）：
   - 完成 AI 检测后，点击侧边栏 **"✍️ 写作建议"**
   - 点击页面上的 **"📥 从检测导入"** 按钮
   - 系统自动导入文本和检测上下文

2. **直接输入**：
   - 在写作建议页面直接粘贴文本
   - 这种方式不含检测上下文，建议先做检测

### 6.2 检测上下文展示

如果从检测结果导入，页面顶部会显示一个**检测上下文面板**：

```
┌─────────────────────────────────────────┐
│  📊 检测上下文                           │
│  风险等级：高风险 (72%)                   │
│  发现 5 个可疑问题                        │
│  已自动选择推荐优化策略                    │
└─────────────────────────────────────────┘
```

### 6.3 选择优化策略

系统提供 5 种优化策略，可以单选也可以组合选择：

| 策略 | 说明 | 自动推荐条件 |
|------|------|-------------|
| **表达改写** (rephrase) | 让语句更自然，降低 AI 模板化特征 | 检测到"表达"或"自然"相关问题 |
| **结构重组** (restructure) | 改变段落组织和论证逻辑 | 检测到"结构"相关问题 |
| **词汇替换** (vocabulary) | 丰富用词，避免过度正式或重复 | 检测到"词汇"相关问题 |
| **语气调整** (tone) | 调整文本的正式程度和个人风格 | 检测到"语气"相关问题 |
| **综合优化** (general) | 全方位改进，适合一般情况 | 默认推荐 |

> 💡 **小技巧**：从检测导入时，系统会根据被标记的具体问题**自动勾选**最相关的策略。例如，如果检测报告指出"语句过于模板化"，系统会自动勾选"表达改写"策略。

### 6.4 获取优化建议

1. 确认文本和策略选择后，点击 **"获取写作建议"** 按钮
2. 系统将利用 AI 分析文本并生成具体的修改建议
3. 等待几秒后，建议列表将出现在页面下方

### 6.5 查看与管理建议

每条建议以**卡片**形式展示，包含：

```
┌──────────────────────────────────────────────┐
│  [表达改写]                     置信度: 85%   │
│  ──────────────────────────────────────────── │
│  原文（红色删除线）：                          │
│  "首先，我们需要考虑多方面的因素。"             │
│                                              │
│  建议修改为（绿色）：                          │
│  "在分析这一问题时，多重因素值得深入探讨。"      │
│                                              │
│  💡 修改说明：                                 │
│  "原句使用了AI常见的'首先…'开头模式，改写后     │
│   采用更自然的表达方式，同时保留了原意。"        │
│                                              │
│              [✅ 接受]  [❌ 拒绝]              │
└──────────────────────────────────────────────┘
```

#### 关键操作：

| 操作 | 说明 |
|------|------|
| ✅ **接受** | 采纳这条建议 |
| ❌ **拒绝** | 不采纳，建议变灰 |
| **全部接受** | 一键接受所有建议 |

### 6.6 预览修改效果

接受建议后，页面下方的**预览区域**会实时显示修改后的文本：

- 被修改的部分以 **绿色高亮** 标识
- 您可以在预览中检查修改是否符合预期
- 风险评分的**预估变化**也会显示

### 6.7 应用修改

1. 确认预览无误后，点击 **"应用选中建议 (N)"** 按钮（N 为已接受的建议数量）
2. 文本框中的内容将被更新为修改后的版本
3. 应用后，您可以：
   - **撤销** (↩️)：恢复到修改前的状态
   - **重新优化**：清除当前建议，选择不同策略重新生成

### 6.8 完整优化流程

```
导入检测文本
    ↓
查看自动推荐策略（可手动调整）
    ↓
获取写作建议
    ↓
逐条审阅 → 接受/拒绝
    ↓
预览修改效果
    ↓
应用建议 → 文本更新
    ↓
（可选）重新检测确认风险已降低
```

> 💡 **最佳实践**：优化后，建议返回 AI 检测页面重新检测一次，确认风险评分已显著降低。

## 7. 复核中心

复核中心是管理和审核检测结果的专业工具，适合教师、编辑或管理员使用。

### 7.1 功能概述

复核中心提供三个标签页：

| 标签页 | 功能 |
|-------|------|
| **待复核** | 显示所有高风险或需要人工确认的检测结果 |
| **申诉** | 处理用户对检测结果提出的异议 |
| **统计** | 查看反馈统计和复核数据 |

### 7.2 待复核列表

**待复核**列表自动收集以下类型的检测结果：

- 风险等级为 **高风险** 或 **极高风险** 的检测
- 复核优先级评分 > 0.5 的检测
- 用户提交了不同意反馈的检测

每条待复核项显示：

| 信息 | 说明 |
|------|------|
| 文本预览 | 被检测文本的前几十字 |
| 风险等级 | 彩色徽章显示风险级别 |
| 时间 | 检测提交的时间 |
| 状态 | 待处理 / 已复核 |

### 7.3 执行复核

1. 点击列表中的任意一条待复核项
2. 查看检测详情和相关证据
3. 选择您的判定：

| 判定 | 含义 |
|------|------|
| **维持** (maintain) | 同意原始检测结论 |
| **调整** (adjust) | 部分修改检测结论 |
| **撤销** (dismiss) | 认为检测结论有误，撤销判定 |

4. 在评论框中填写您的复核意见（建议详细说明理由）
5. 点击 **"提交复核"** 按钮

### 7.4 统计信息

统计标签页展示：

- **总反馈数**：平台收到的用户反馈总量
- **认同率**：用户对检测结果表示认同的比例
- **待处理数**：当前未复核的项目数量
- **申诉数**：待处理的用户申诉数量

## 10. 常见问题解答 (FAQ)

### 关于检测

#### Q：检测结果说我的文章是 AI 写的，但明明是我自己写的？

**A**：这种情况（假阳性）是可能发生的。以下情况容易被误判：
- 使用了大量学术惯用句式（如"综上所述"、"由此可见"等）
- 文本高度结构化、格式化
- 专业术语密集的技术类文章
- 先用 AI 翻译再修改的文本

建议：
1. 使用**写作建议**功能增加个人化表达
2. 通过**反馈**告知系统误判，帮助改进
3. 如果是课程作业，可以向教师说明并提供写作过程证据

#### Q：检测需要多长时间？

**A**：通常 5 秒到 2 分钟不等，取决于文本长度和系统负载。如果超过 3 分钟仍未出结果，建议刷新页面重试。

#### Q：最少需要多少文字才能检测？

**A**：至少 **200 个字符**。文本越长，检测结果越准确。建议至少提交一个完整段落（300 字以上）。

#### Q：支持哪些语言？

**A**：目前主要支持 **中文** 和 **英文**。系统可以自动检测语言，也可以手动指定。

#### Q：一次能检测多长的文章？

**A**：系统对文本长度没有严格的上限限制，但过长的文章（超过 20000 字）建议分段检测以获得更精确的结果。

### 关于写作建议

#### Q：使用写作建议修改后，是否保证通过检测？

**A**：不保证，但通常可以显著降低风险评分。建议修改后重新检测确认。如果仍然偏高，可以尝试不同的优化策略或手动进一步修改。

#### Q：写作建议会改变我的文章意思吗？

**A**：系统会尽力保持原意不变，仅在表达方式上做调整。但建议您**逐条审阅**每一条修改建议，确认是否符合您的表达意图后再接受。

#### Q：可以只接受部分建议吗？

**A**：可以。每条建议都有独立的"接受"和"拒绝"按钮，您可以选择性地采纳。只有被标记为"已接受"的建议才会在应用时生效。

### 关于账号

#### Q：忘记密码怎么办？

**A**：目前版本暂不支持自助密码重置。请联系系统管理员帮助重置密码。

#### Q：可以修改用户名或邮箱吗？

**A**：目前版本暂不支持在界面上修改。如需修改，请联系管理员。

### 关于隐私

#### Q：我提交的文本会被保存吗？

**A**：检测记录会保存在系统数据库中，用于提供历史查询功能。检测结果缓存 24 小时后自动清理。系统不会将您的文本用于模型训练或分享给第三方。

#### Q：其他人能看到我的检测结果吗？

**A**：普通用户只能看到自己提交的检测记录。管理员和复核人员可以在复核中心查看高风险的检测结果，但无法查看其他用户的完整检测历史。

## 11. 术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| AI 检测 | AI Detection | 识别文本是否由人工智能生成的技术 |
| 风险等级 | Risk Level | 文本呈现 AI 特征的程度分类 |
| 风险评分 | Risk Score | 0-100% 的量化评分，越高表示 AI 特征越明显 |
| LLM | Large Language Model | 大语言模型，如 GPT、Claude、Qwen 等 |
| 置信度 | Confidence | 模型对其判断的确信程度 |
| 统计特征 | Statistical Features | 通过数学方法计算的文本量化指标 |
| 证据完整度 | Evidence Completeness | 检测过程中收集到的证据是否充分 |
| 热力图 | Heatmap | 用颜色深浅表示各段落风险程度的可视化图表 |
| 标注文本 | Highlighted Text | 在原文中以高亮方式标记可疑片段 |
| 假阳性 | False Positive | 将人类原创文本误判为 AI 生成 |
| 假阴性 | False Negative | 未能检出实际为 AI 生成的文本 |
| 写作建议 | Writing Suggestions | 针对 AI 特征提供的文本改写建议 |
| 复核 | Review | 对高风险检测结果进行人工确认的过程 |
| 检测粒度 | Granularity | 检测的精细程度（文档级/段落级/句子级） |
| 降级模式 | Degradation Mode | 主模型不可用时的备用检测方式 |
